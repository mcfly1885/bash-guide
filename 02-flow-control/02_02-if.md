# If Instruction

The instruction `if` is used to determine what the script have to do **if** a specific condition occur.

Syntax:
```bash
if [[ condition ]]; then
    my code here
fi
```
The code inside the if block will be execute only if the condition is true.

Note: respect the spaces between the square brackets and the condition, otherwise bash throw a syntax error. 
```bash
#this works
if [[ condition ]]; then
    #my code here
fi 

#this doesn't work
if [[condition]]; then
    #my code here
fi

#neither this will work...
if [[ condition]]; then
    #my code here
fi

#or 
if [[condition ]]; then
    #my code here
fi
```

### Example
```bash
#!/bin/bash

age=22
if [[ $age -ge 21 ]]; then
    echo "you are adult"
fi
```
In this example the code will echo the string "you are adult" because the condition declared is true.

Let's try to read the code like this:
```
age equal to 22
if age is greater then 21 print the string "you are adult"
```
Is possible determinate what the script have to do also if the specified condition is **FALSE** with the instruction `else`. 

Let's improve the script:
```bash
#!/bin/bash

age=18
if [[ $age -ge 21 ]]; then
    echo "you are adult"
else
    echo "you are NOT adult"
fi
```
and try to read the code:
```
age equals to 18
if age is greater then 21 print the string "you are adult"
else print the string "you are not adult"
```

In the second version of script, the code will output "you are NOT adult" because the variable age is less then 18, so the condition il false.


The `else` instruction is facultative.


## Multiple Condition

In Bash is possible combine multiple condition with Logic Operators.


```bash
[[ condition1 && condition2 ]] 
# logic AND, return true if all the condition specificated are true.

if [[ 7 -gt 6 && 4 -lt 5 ]]; then
...
fi

[[ condition1 || condition2 ]]
# logic OR, return true if at least one condition specificated is true. Return false only if both condition are false. 

if [[ 7 -gt 6 || 4 -le 5 ]]; then
...
fi

[[ ! condition ]] 
# logic NOT, reverse the condition result: if it's true become false, if is false become true.

if [[ ! 7 -lt 6 ]]; then #true
...
fi

#looking at the example above the question is: 
#"Is true that 7 is NOT less then 6?" in this case the expression return true.
```

## Multiple if conditions

It's possible define **multiple if conditions** in two ways:

### Through the key word `elif`

This method equals to set a second condition if the first is false.

```bash
if [[ condition1 ]]; then
    #my code here
elif [[ condition2 ]];then
    #my code here
else
    #my code here
if
```
The code inside the `elif` block will be executed only if *condition1* is **false** and *condition2* is **true**.

If both conditions are false, the if block execute the code inside `else` block. The `else` instruction is not mandatory.

## Example

```bash
myfile="path/to/my/file"

if [[ ! -e $myfile ]]; then

    echo "the path does not exist"
    #if condition is true the if block ends here

elif [[ -f $myfile ]]; then

    echo "something exists on path and it's a regular file"
    #if the first condition is false the if block ends here

else

    echo "something exists on path but it's not a file"
    #if all conditions are false the if block ends here.

fi

```
### Through Nested If
The code inside the if blocks is executed if both are true

```bash
if [[ condition1 ]]; then
	if[[ condition2 ]]; then
        # my code here
	fi
fi 
```

```bash
name="john"
age="21"

if [[ $age -ge 21 ]]; then
	if[[ $name="john" ]]; then
        echo "Hello John"
        #this if block will be execute only both condition
        #are true
	fi
fi 
```

## Best Practices
- The if-elif block is a good practice if the conditions are less than 3 or 4. Avoid to add too many elif blocks can help to keep the code in order and more readable.
- If the condition are greater than 3 or 4 valutate to use the [case](/bash-guide/work-in-progress.md) block.
